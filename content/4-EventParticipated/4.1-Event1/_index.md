# Summary Report “FCJ Community Day - AI & Prompt Engineering”

### Event Overview

The event explored how AI is changing learning, career development, and software delivery. In addition to Prompt Engineering, the sessions discussed attention and motivation, professional growth, AWS architectures for AI applications, and the use of AI Agents within a structured Software Engineering process.

### Main Topics

#### 1. Designing learning through fast feedback

**Huynh Hoang Long** explained how social platforms maintain attention through dopamine, instant feedback, short-term rewards, and fear of missing out. These mechanisms can be adapted positively to learning by dividing large goals into smaller tasks, making progress visible, and creating frequent feedback after each step.

#### 2. Improving AI output with Prompt Engineering

**Nguyen Tuan Thinh** presented **“Automated Prompt Engineering: Enhancing LLM Output Quality.”** An effective prompt should clearly define:

- The AI's role;
- The required task;
- Context and input data;
- Reference examples;
- Expected output format;
- Constraints and evaluation criteria.

The discussion on **Token Economics** showed that a better prompt can improve accuracy while reducing token usage and operating cost. The **Proptimizer** tool was also introduced as a way to review and refine prompts.

#### 3. Building AI applications on AWS

A typical AI solution was illustrated through services such as **Amazon API Gateway, AWS Lambda, Amazon Cognito, Amazon Bedrock, Amazon DynamoDB**, and **Amazon CloudWatch**. This provided a complete view of the flow from receiving requests and authenticating users to invoking an AI model, storing data, and monitoring the application.

#### 4. Professional growth in the AI era

**Khang** discussed growth mindset, proactive learning, asking “Why?”, and integrity at work. AI can accelerate tasks, but the final quality still depends on critical thinking, verification, and the user's responsibility for the result.

#### 5. Using AI Agents in software development

**Thao** presented **“Don’t Just Vibe Code – Build Software Engineering Standards.”** Instead of using AI only to generate code, the BMAD approach assigns specific responsibilities:

- **PM Agent** supports product requirements;
- **Architect Agent** proposes the system design;
- **Scrum Master Agent** organizes Epics and Stories;
- **Developer Agent** implements features;
- **Review Agent** tests, reviews, and improves the codebase.

This approach demonstrates that AI provides greater value when it operates inside a controlled process with clear quality standards.

### Knowledge and Skills Gained

- Understood the relationship between prompt quality and AI response quality.
- Learned how to create prompts with clear context, structure, and constraints.
- Recognized the need to verify AI-generated content and remain responsible for the final decision.
- Gained an overview of an AWS-based AI application architecture.
- Understood how AI Agents can support requirements, design, coding, testing, and review.
- Learned that effective study plans need small goals, regular feedback, and measurable progress.

### Practical Application

- Standardize prompts when using AI for requirements analysis, documentation, or programming support.
- Review AI-generated results before applying them to a project.
- Experiment with assigning different AI roles during software projects.
- Continue studying Bedrock, Lambda, Cognito, DynamoDB, and CloudWatch.
- Break work into smaller tasks and track results to maintain learning momentum.

### Event Reflection

The event provided a balanced view of technology and human responsibility. It did not only show how to use AI more effectively, but also emphasized process discipline, professional behavior, and quality control. The main lesson was that AI can improve productivity, while analysis, verification, and final decision-making must remain under human control.
