**On this page**, you will need to introduce your worklog. **How** did you complete it? How many weeks did you take to complete the program? **What** did you do in those weeks?

Typically, and as a standard, a worklog is carried out over about 3 months (throughout the internship period) with weekly contents as follows:

**Week 1:** [Getting familiar with the program and learning AWS core services, AWS Budgets, AWS Support, and Kiro](1.1-week1/)

**Week 2:** [Learning Amazon VPC, network security, VPC Peering, Transit Gateway, VPN, and Hybrid DNS](1.2-week2/)

**Week 3:** [Learning Amazon EC2, IAM, resource management, monitoring, load balancing, and Auto Scaling](1.3-week3/)

**Week 4:** [Learning Amazon S3, CloudFront, Storage Gateway, AWS Backup, SNS, and Amazon FSx](1.4-week4/)

**Week 5:** [Learning advanced AWS IAM, IAM Roles, ABAC, identity governance, and security monitoring](1.5-week5/)

**Week 6:** [Learning Permission Boundaries, AWS KMS, CloudTrail, Athena, SQL, and AWS database services](1.6-week6/)

**Week 7:** [Consolidating AWS knowledge in architecture, security, storage, databases, and monitoring](1.7-week7/)

**Week 8:** [Analyzing requirements, designing the architecture, and initializing the Payment Gateway project](1.8-week8/)

**Week 9:** [Implementing Account Service, Transaction Service, and idempotency](1.9-week9/)

**Week 10:** [Implementing Payment Service, distributed locking, resilience, and rate limiting](1.10-week10/)

**Week 11:** [Integrating the system and deploying AWS services](1.11-week11/)

**Week 12:** [Running integration and load tests, optimizing, and finalizing the project](1.12-week12/)
